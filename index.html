The Blockchain

An append-only log of every transaction that has moved Bitcoin currency from one address to another in the Bitcoin system.

You're more likely to hear ledger instead of log.

Bitcoin as a Distributed System

It's not a distributed system for the sake of…

  • scaling storage capacity
  • scaling computation

It is a distributed system for the sake of…

  • redundancy and fault tolerance
  • anonymity
  • decentralized ownership

P2P Node Discovery

  • First, check actively-maintained DNS records
  • Query IRC channels
  • Worst case scenario, use hardcoded list of nodes that were known active.
Similar to gnutella and other p2p networks.

The P2P Network

From the network, a Bitcoin node gleams:

  • A list of IP addresses known to peers.
  • Broadcasted transactions that are valid based on its current/main copy of the blockchain.

A full node also maintains:

  • A copy of the blockchain.

Consensus

The consensus algorithm attempts to solve two problems:

  • What transactions are committed to the blockchain?
  • How does new coin get injected into the system?
Similar to gnutella and other p2p networks.

Consensus: What transactions are committed to the blockchain?

We could use the leader/follower pattern of Raft.

But we want to inject new coins in a way that rewards actors contributing to the system.

We can reward actors that validate and commit transactions to the blockchain.

In Raft, we invite race-to-leader abuse when being a leader is reward-able.

So instead of leader election, we have mining.

A miner:

  1. Asks a node for a block of uncommitted valid txs.
  2. Adds a reward-me tx of its own to the block.
  3. Performs proof-of-work that signs block as confirmed.
  4. If able to successfully confirm block, has node broadcast it as part of blockchain.

Proof-of-Work

The bulk of the work is artificially difficult as a way to throttle the system: block_hash = SHA256(SHA256(EndianFlipForEach32Bits(First80BytesOf(data and nonce)))) if block_hash < difficulty_target: tell everyone about our new block in the blockchain exit and try to generate/mine another block else: nonce = nonce + 1 block_hash = SHA256(SHA256(EndianFlipForEach32Bits(First80BytesOf(data and nonce))))

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the background.

Custom Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
